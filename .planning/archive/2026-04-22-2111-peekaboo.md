## User Intent
Peekaboo is a cozy find-the-hidden-character game where a fog-of-war covers a scene and the player clicks to reveal cells until they find the target. Each round introduces a character (from a rotating emoji cast) on a "meet" screen, then the character flies into the scene and vanishes ‚Äî the player sees the environment briefly but never the target's hiding spot. Fog rolls in, and the player pokes holes in it by clicking cells until they uncover the target, triggering a celebration. Difficulty stays flat across rounds, and all transitions are click-to-proceed (no timed auto-advance) for accessibility.

  12. **Theme support**: dark/light via existing theme system. Fog color adapts (darker fog in light theme, lighter fog in dark theme, or vice versa for contrast).

### What Didn't
1. **Blocker: `showScreen` didn't toggle `.active` class** (renderer.ts). Shared `GameScreen` requires `.active` for `visibility: visible; transform: none`. Peekaboo only set `data-active-screen`. All screens past meet were invisible. Squares/waterwall both toggle `.active`.
2. **High: CSS/HTML class name mismatch.** CSS defined `.peekaboo-btn`, `.peekaboo-playing-container`, `.peekaboo-enter-scene`, `.peekaboo-round-label` but HTML used `.peekaboo-primary-btn`, `.peekaboo-screen-panel`, `.peekaboo-heading-block`, `.peekaboo-kicker`, `.peekaboo-title`, `.peekaboo-game-body`, `.peekaboo-menu-btn`, `.peekaboo-enter-body`, `.peekaboo-found-card`, etc. Most elements rendered with browser defaults ‚Äî buttons, panels, headers all unstyled.
3. **High: Scene layer had no positioned parent.** CSS defined `.peekaboo-playing-container { position: relative }` but HTML used `.peekaboo-game-body` with no CSS. Scene layer (`position: absolute; inset: 0`) had no containing block.
4. **High: Missing animation CSS.** `animations.ts` added `.peekaboo-cell--revealing` and `.peekaboo-cell--fog` classes with no CSS rules ‚Äî animations were visual no-ops.
5. **High: Enter screen missing scene element.** CSS defined `.peekaboo-enter-scene` but HTML didn't include this element. Enter screen had no visual scene background.
