Create `src/super-word/renderer.ts`. Import `GameState`, `Puzzle`, `SceneItem` from `./types.js`. The renderer is a pure side-effect layer — it reads state and updates the DOM. It MUST NOT attach event listeners (that is input.ts's job) and MUST NOT mutate game state.

Export these functions:

**`renderScene(puzzle: Puzzle, state: GameState, sceneEl: HTMLElement): void`**
- Clear sceneEl innerHTML
- For each item in puzzle.items:
  - Create a `<button>` with class `scene-item`, `data-item-id="{item.id}"`, `data-item-type="{item.type}"`, style `left: {item.x}%; top: {item.y}%`
  - If item.type is 'letter': also set `data-char="{item.char}"`
  - aria-label: letter items = `"{item.emoji} {item.label} — contains letter {item.char}"`, distractors = `"{item.emoji} {item.label}"`
  - Inner `.item-card` div containing emoji span (class `item-emoji`, font-size from CSS) and letter badge span (class `item-badge`, only for letter items — shows the char in a colored circle, background color from `--game-badge-{index % 5}`)
  - If item is collected (state.collectedLetters has sourceId === item.id): add class `collected` (CSS: display: none)
  - Roving tabindex: First uncollected item gets `tabindex="0"`, all others get `tabindex="-1"`

**`renderLetterSlots(state: GameState, puzzle: Puzzle, slotsEl: HTMLElement): void`**
- Clear slotsEl innerHTML
- Total slots = puzzle.answer.length
- For each slot index 0..totalSlots-1:
  - If index < state.collectedLetters.length: create `<div>` with class `letter-tile`, `role="option"`, `tabindex="0"`, `data-index="{index}"`, text content = letter char, `aria-label="{char}, position {index+1} of {total}"`. If index === state.selectedTileIndex, add class `selected`.
  - Else: create `<div>` with class `empty-slot`, `role="option"`, `aria-disabled="true"`, `aria-label="Empty slot {index+1} of {total}"`

**`renderGameHeader(state: GameState, puzzle: Puzzle, puzzleIndex: number, totalPuzzles: number): void`**
- Set `#puzzle-counter` textContent to `"{puzzleIndex + 1} / {totalPuzzles}"`
- Set `#prompt-text` textContent to `puzzle.prompt`
- Set `#score` textContent to `"⭐ {state.score}"` and update `aria-label` to `"Score: {state.score}"`
- Set `#letters-count` textContent to `"{state.collectedLetters.length} / {puzzle.answer.length}"`

**`showScreen(screenId: string): void`**
- Remove class `active` from all elements with class `screen`
- Add class `active` to element with id `screenId`

**`showToast(message: string, duration: number): void`**
- Set `#feedback-toast` textContent to message
- Add class `show` to `#feedback-toast`
- setTimeout: remove class `show` after `duration` ms

**`renderCompleteScreen(puzzle: Puzzle, state: GameState): void`**
- Set `#solved-word` innerHTML to individual `<span>` per letter in puzzle.answer, each with class `solved-letter` and `style="animation-delay: {i * 0.1}s"` for staggered animation

**`renderWinScreen(state: GameState): void`**
- Set `#final-score` textContent to `"⭐ {state.score}"`

**`setCheckButtonEnabled(enabled: boolean): void`**
- Get `#check-btn` element
- If enabled: remove `disabled` attribute, remove class `disabled`
- Else: set `disabled` attribute, add class `disabled`

All DOM queries use `document.getElementById()`. Cache element references at module scope if queried more than once (lazy initialization pattern — query on first call, store in module-level variable).

**Export `InputCallbacks` interface:**
```typescript
export interface InputCallbacks {
  onLetterCollected: (item: SceneItem) => void
  onDistractorClicked: (item: SceneItem) => void
  onTileSelected: (index: number) => void
  onLettersSwapped: (indexA: number, indexB: number) => void
  onCheckAnswer: () => void
  onHintRequested: () => void
  onNextPuzzle: () => void
  onStartGame: () => void
  onPlayAgain: () => void
}
```

**Export `setupInput(getState: () => GameState, setState: (s: GameState) => void, getPuzzle: () => Puzzle, callbacks: InputCallbacks): void`**

This function attaches all event listeners once. It reads state via `getState()` and writes via `setState()`. The callbacks notify main.ts of events for accessibility announcements and animations.

**Scene item interaction (Pointer Events — GAME-03):**
- Attach delegated `pointerdown` listener on `#scene` container (not individual items — items are re-rendered)
- On `pointerdown` on a `.scene-item`: call `e.preventDefault()` to prevent touch scrolling
  - Read `data-item-type` and `data-item-id` from clicked element (traverse up to `.scene-item` if needed)
  - If `data-item-type === 'letter'` and not already collected: read `data-char`, call `callbacks.onLetterCollected(item)` — main.ts handles state update and rendering
  - If `data-item-type === 'distractor'`: call `callbacks.onDistractorClicked(item)`
- Apply `touch-action: none` on `#scene-wrapper` via `sceneWrapper.style.touchAction = 'none'`

**Letter tile interaction — click-to-swap (Pointer Events):**
- Attach delegated `pointerdown` listener on `#letter-slots` container
- On `pointerdown` on a `.letter-tile`:
  - Read `data-index` as number
  - Store pointer start position (clientX, clientY) and the index
  - If no tile selected in state: call `callbacks.onTileSelected(index)`
  - If another tile already selected: call `callbacks.onLettersSwapped(selectedIndex, index)`
  - If same tile: call `callbacks.onTileSelected(index)` (toggles deselect)

**Letter tile interaction — drag-to-reorder (Pointer Events):**
- On `pointerdown` on `.letter-tile`: `tile.setPointerCapture(e.pointerId)`, record start position
- On `pointermove` (with capture): if moved > 4px from start, enter drag mode
  - Create ghost clone: clone the tile, append to game container with `position: fixed`, `pointer-events: none`, `opacity: 0.8`, `z-index: 50`
  - Update ghost position to follow `e.clientX`, `e.clientY`
  - Add `.dragging` class to original tile (opacity 0.45 per UI-SPEC)
- On `pointerup`:
  - If in drag mode: determine drop target via `document.elementFromPoint(e.clientX, e.clientY)`, find closest `.letter-tile` target
  - If valid target with different index: call `callbacks.onLettersSwapped(sourceIndex, targetIndex)`
  - Remove ghost element, remove `.dragging` class, release pointer capture
- On `pointercancel`: clean up ghost, restore tile state

**Keyboard navigation — scene items (roving tabindex — GAME-02):**
- Attach `keydown` listener on `#scene` container
- Arrow keys (ArrowUp, ArrowDown, ArrowLeft, ArrowRight): find nearest uncollected scene item in that direction. Use a `findNearestInDirection` helper that:
  - Gets all uncollected `.scene-item` elements and their x,y from `data-x`/`data-y` or computed position
  - Filters items in the correct quadrant relative to current focused item
  - Picks closest by Euclidean distance
  - Calls `.focus()` on target and updates tabindex (set target to `tabindex="0"`, current to `tabindex="-1"`)
- Enter or Space (`key === 'Enter' || key === ' '`): trigger the same action as pointerdown (collect if letter, shake if distractor). Call `e.preventDefault()` to prevent page scroll on Space.
- When an item is collected and removed from scene: move focus to nearest remaining uncollected item

**Keyboard navigation — letter tiles (GAME-02):**
- Attach `keydown` listener on `#letter-slots` container
- ArrowLeft: swap focused tile with left neighbor (index - 1). Call `callbacks.onLettersSwapped(currentIndex, currentIndex - 1)`. After re-render, focus the tile at the new position.
- ArrowRight: swap focused tile with right neighbor (index + 1). Call `callbacks.onLettersSwapped(currentIndex, currentIndex + 1)`. After re-render, focus the tile at the new position.
- Enter or Space: toggle click-to-swap selection. Call `callbacks.onTileSelected(index)`.
- Prevent default on Arrow keys to stop page scrolling.

**Button handlers:**
- `#start-btn` `pointerdown` or `click` → `callbacks.onStartGame()`
- `#check-btn` `pointerdown` or `click` → `callbacks.onCheckAnswer()` (only if not disabled)
- `#hint-btn` `pointerdown` or `click` → `callbacks.onHintRequested()`
- `#next-btn` `pointerdown` or `click` → `callbacks.onNextPuzzle()`
- `#replay-btn` `pointerdown` or `click` → `callbacks.onPlayAgain()`

Use `click` events for buttons specifically (they work correctly with both pointer and keyboard activation). Use `pointerdown` only for scene items and tiles where touch/drag behavior matters.

**CRITICAL: No `mousedown`, `touchstart`, or `dragstart` events anywhere.** Only `pointerdown`/`pointermove`/`pointerup`/`pointercancel` for scene/tile interactions, and standard `click` for buttons.
