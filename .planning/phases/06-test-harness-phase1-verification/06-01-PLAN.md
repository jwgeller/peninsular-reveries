1. Install devDependencies:
   - `@playwright/test` (latest)
   - `serve` (latest — used as local static file server per D-01)

2. Add `"test"` script to package.json scripts:
   ```json
   "test": "npx playwright test"
   ```

3. Create `playwright.config.ts` at project root:
   ```typescript
   import { defineConfig } from '@playwright/test';

   export default defineConfig({
     testDir: './tests',
     fullyParallel: true,
     forbidOnly: !!process.env.CI,
     retries: process.env.CI ? 2 : 0,
     workers: process.env.CI ? 1 : undefined,
     reporter: 'html',
     use: {
       baseURL: 'http://localhost:3000',
       trace: 'on-first-retry',
     },
     webServer: {
       command: 'npx serve dist -l 3000',
       port: 3000,
       reuseExistingServer: !process.env.CI,
     },
   });
   ```
   Per D-01: Playwright's built-in `webServer` config starts/stops `serve` automatically — `npm test` is a single command.

4. Run `npx playwright install --with-deps chromium` to install browser binary (chromium only — sufficient for structural tests).

5. Verify: `npm test` runs and exits (may have 0 tests found, that's fine — test files come in next task).
  </action>

    - playwright.config.ts contains `webServer` block with `command: 'npx serve dist -l 3000'`

**tests/site-01-responsive.spec.ts** (SITE-01 — Responsive layout):
- Define 3 viewports: phone (375×667), tablet (768×1024), desktop (1280×800)
- For each viewport, navigate to homepage (`/`) and game page (`/super-word/`)
- Assert: page loads without console errors (attach console error listener)
- Assert: `<main>` element is visible
- Assert: no horizontal overflow (`document.documentElement.scrollWidth <= document.documentElement.clientWidth + 1`)

**tests/site-02-navigation.spec.ts** (SITE-02 — Navigation):
- Test: Homepage has link to Super Word game page (`a[href*="super-word"]`)
- Test: Clicking game link navigates to `/super-word/` (URL contains `super-word`)
- Test: Game page has navigation back to homepage (link with `href` containing `./` or `/`)
- Test: Browser back button returns to previous page after navigating forward
- Test: All 3 pages are directly URL-addressable — navigate to `/`, `/super-word/`, `/404.html` and verify each returns 200 (or rendered content for 404)

**tests/site-03-semantic-html.spec.ts** (SITE-03 — Semantic HTML):
- For each page (`/`, `/super-word/`, `/404.html`):
  - Assert: `<main>` element exists
  - Assert: `document.querySelector('nav') !== null` OR `document.querySelector('[role="navigation"]') !== null` OR page has header with navigation links (shell.ts injects nav into `#site-header`)
  - Assert: `<h1>` exists (heading hierarchy starts at h1)
  - Assert: `document.querySelector('meta[name="description"]')` exists and has non-empty `content`
  - Assert: `<title>` is non-empty
